# 实验报告：Actor-Critic 与 DDPG 在倒立摆上的仿真与分析

> 实验环境：Pendulum-v1 (Gymnasium) | 系统 Python 3.12 | PyTorch 2.8 | 2026-05-30

---

## 1. Reward 机制说明

### 1.1 环境 Reward 公式（Pendulum-v1）

Pendulum-v1 的真实 reward 由 Gymnasium 内置公式定义：

$$r_{\text{true}} = -(\theta^2 + 0.1 \cdot \dot{\theta}^2 + 0.001 \cdot \tau^2)$$

其中：
- $\theta$：摆杆角度（$\theta=0$ 表示倒立平衡，即目标状态）
- $\dot{\theta}$：角速度
- $\tau$：施加在摆杆上的力矩（控制输入），范围 $[-2, 2]$

reward 始终为负值，**最大化 reward 等价于让倒立摆尽可能维持在 $\theta\approx0$ 且 $\dot{\theta}\approx0$，同时减少控制能耗**。最优策略下 reward 约 $-120$ 左右。

### 1.2 延迟 Reward 机制

项目在 `envs.py` 的 `DelayedRewardPendulum` 类中实现了两种模式：

- **无延迟**（`delay=0`）：`delayed_reward = true_reward`，agent 直接收到当前步的真实环境 reward
- **有延迟**（`delay=5`）：维护一个长度为 5 的 FIFO 队列，每步将 `(state_before_action, action)` 入队。当队列长度超过 5 时，弹出 5 步前的记录，用那个时刻的 $(\theta, \dot{\theta}, \tau)$ 重新计算 reward 返回给 agent。即：**agent 在第 $t$ 步收到的 reward 实际来自第 $t-5$ 步的状态和动作**

`info["true_reward"]` 始终保存当前步的真实 reward，用于评估指标。

## 4. 各算法详细分析

### 4.1 Actor-Critic 无延迟

| 指标 | 值 |
|------|------|
| 最终 episodic reward | -1526.39 |
| 最终 smoothed true reward | -1240.14 |
| 最佳 smoothed true reward | -1170.03 |
| 收敛情况 | 120 个 episode 后仍未收敛 |

**分析**：Actor-Critic 在整个 120 个 episode 中 reward 大幅波动（-900 ~ -1550），没有明显的下降收敛趋势。原因在于：

1. **On-policy 高方差**：基础的 Advantage Actor-Critic 使用 TD error $\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$ 作为优势估计，单步 TD 本身方差较大
2. **连续动作空间的挑战**：$(\theta, \dot{\theta})$ 连续状态空间使得策略梯度估计的方差进一步放大
3. **缺乏经验回放**：与 DQN/DDPG 不同，Actor-Critic 是 on-policy 的，每个 transition 只用一次就被丢弃，样本效率低

### 4.2 Actor-Critic 延迟 5

| 指标 | 值 |
|------|------|
| 最终 episodic reward (delayed) | -1485.13 |
| 最终 true reward | -1528.95 |
| 最终 smoothed true reward | -1261.85 |

**分析**：加入 5 步延迟后，Actor-Critic 表现进一步恶化。

- 延迟导致 agent 收到的 reward 与当前动作的因果关系被严重削弱——它学到的是"5 步前的状态+动作"的价值，而非当前策略的真实价值
- Actor-Critic 的梯度直接依赖 TD error，延迟使 TD target 中的 $r_t$ 与当前 $(s_t, a_t)$ 不匹配，产生误导性梯度
- 与无延迟相比，true reward 甚至略差（-1528.95 vs -1526.39），说明延迟不仅没有帮助，还引入了额外的噪声

### 4.3 DDPG 无延迟 ⭐

| 指标 | 值 |
|------|------|
| 最终 episodic reward | -119.04 |
| 最终 smoothed true reward | -118.36 |
| 达到 -130 以下的 episode | 第 31 个 episode |
| 完全收敛 episode | 第 40 个 episode |
| 收敛后稳态均值 | ≈ -120 |

**分析**：DDPG 在无延迟下表现优异，收敛过程清晰：

- **第 1-15 个 episode**（warmup 阶段，前 1000 步纯随机探索）：reward 在 -1100 ~ -1800 大幅波动
- **第 15-30 个 episode**：从经验池采样学习后快速下降，reward 从 -1000 迅速降至 -120
- **第 31 个 episode 起**：reward 达到 -120 附近（Pendulum-v1 的最优水平），策略基本收敛
- **第 40 个 episode 后**：完全稳定在 -118 ~ -122，波动极小

DDPG 成功的关键设计：
1. **经验回放**：打破时序相关性，提高样本效率
2. **目标网络 + 软更新**：$\theta_{\text{target}} \gets \tau\theta_{\text{source}} + (1-\tau)\theta_{\text{target}}$，提供稳定的学习目标
3. **确定性策略梯度**：通过 Critic $Q(s,a)$ 的梯度直接指导 Actor 更新，学习效率高
4. **OU 噪声**：在确定性策略上添加探索噪声，实现高效的连续空间探索

### 4.4 DDPG 延迟 5

| 指标 | 值 |
|------|------|
| 最终 episodic reward (delayed) | -124.53 |
| 最终 true reward | -124.54 |
| 最终 smoothed true reward | -125.86 |
| 开始显著收敛的 episode | 第 53 个 episode |
| 达到 -130 以下的 episode | 第 60 个 episode |
| 收敛后稳态均值 | ≈ -125.9 |

**分析**：延迟 5 对 DDPG 的影响体现在三个方面：

**① 收敛速度明显变慢**
- 无延迟在第 31 个 episode 就达到 -120，延迟 5 要到第 53 个 episode 才开始显著下降
- 第 30-52 个 episode 之间，reward 在 -1500 和 -120 之间反复跳跃，agent 经历了更长的探索期

**② 最终性能略有下降**
- 无延迟稳态均值 ≈ -118.36，延迟 5 稳态均值 ≈ -125.86，性能下降约 6%
- 这是因为延迟 reward 引入了 Q 值估计的偏差，导致学到的策略略逊于最优

**③ 但最终仍然成功收敛**
- 说明 DDPG 的经验回放机制对延迟 reward 具有天然鲁棒性
- 经验池中的 transition 虽然使用延迟 reward 存储，但目标网络的软更新使 Q 函数能逐步逼近真实值函数

## 5. DDPG vs Actor-Critic 对比总结

| 维度 | Actor-Critic | DDPG |
|------|:------------:|:----:|
| 无延迟收敛 | ❌ 不收敛 | ✅ 约 40 episode 收敛 |
| 延迟 5 收敛 | ❌ 不收敛 | ✅ 约 60 episode 收敛 |
| 收敛后性能 (delay=0) | N/A | -118.36 |
| 收敛后性能 (delay=5) | N/A | -125.86 |
| 延迟导致性能损失 | — | 约 6.3% |
| 对延迟敏感度 | **高**（完全无法学习） | **中**（收敛变慢，性能略降） |

### 5.1 为什么 DDPG 对延迟更鲁棒？

1. **Off-policy 学习**：从经验池中采样过去的 transition 进行学习，不要求数据来自当前策略。延迟 reward 被存储后仍可作为训练信号
2. **目标网络机制**：Q 网络的更新目标由目标网络生成，目标网络的参数变化缓慢（soft update），避免学习目标因延迟 reward 而剧烈震荡
3. **Critic 的泛化能力**：Q 函数 $Q(s,a)$ 本质上是拟合一个函数，通过大量经验可以学习到"虽然 reward 有延迟，但平均而言仍然反映了状态-动作的价值"

### 5.2 为什么 Actor-Critic 失败？

1. **On-policy 更新的限制**：每一步的 TD error 都直接用于策略梯度，延迟 reward 使 TD target 中的 $r_t$ 与当前 $(s_t, a_t)$ 不匹配
2. **高方差放大**：单步 TD(0) 本身方差大，延迟进一步放大了方差，使梯度方向可信度降低
3. **无经验回放**：每个 transition 只用一次，无法像 DDPG 那样从历史经验中重新学习

---

## 6. 结论

1. **DDPG 在 Pendulum-v1 上能成功学到最优策略**（reward ≈ -120），与在线教程结果一致
2. **基础的 Actor-Critic 在 Pendulum-v1 上难以收敛**，reward 持续在 -1000 ~ -1500 大幅波动
3. **5 个采样周期的 reward 延迟对 DDPG 的影响**：收敛速度减慢约 50%，最终性能下降约 6%，但最终仍能收敛
4. **DDPG 的 off-policy 特性（经验回放 + 目标网络）使其对 reward 延迟具有天然鲁棒性**，是处理延迟 reward 问题的有效基线方法
5. **Actor-Critic 对 reward 延迟极为敏感**，在延迟环境下完全无法学习
